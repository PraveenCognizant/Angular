Components

What is a Component?

A component = HTML (template) + CSS (styles) + TypeScript (logic) bundled together.

@Component({
  selector: 'app-root',       // ← HTML tag name: <app-root>
  imports: [RouterOutlet],    // ← other Angular pieces this needs
  templateUrl: './app.html',  // ← HTML template
  styleUrl: './app.css'       // ← CSS file
})
export class App { ... }   // ← TypeScript class with logic

@Input() and @Output()

@Input() → Parent passes data DOWN to child (like function arguments)

@Output() → Child sends events UP to parent (like callbacks)

<!-- Parent template: passing data down, listening for events up -->
<app-counter-child
  [message]="parentMessage"    <!-- @Input() -->
  [count]="parentCount()"      <!-- @Input() -->
  (clicked)="handleChildClick($event)">  <!-- @Output() -->
</app-counter-child>

Parent count (updated by child clicks): {{ parentCount() }}

Lifecycle Hooks Order (open browser console)

  1. ngOnChanges() — fires when any @Input value changes (before init)
  2. ngOnInit() — fires once after first render → use for API calls
  3. ngAfterViewInit() — fires after template renders → use for DOM access
  4. ngOnDestroy() — fires before removal → clean up here

Interview Tips

  • Always use ngOnInit for HTTP calls, not the constructor
  • Constructor is only for dependency injection (inject services)
  • @Output uses EventEmitter — always emit() to send data
  • Child must be in parent's imports: [] array to be used
